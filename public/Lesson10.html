Version Control/Git

  • saves the state of code at different times and allows the user to go back to that "commit"
  • branches allow you to work on a different verion of your project, create or test big parts of the code can be branched
  • distributed: can work on the code separately. One person can do CSS, someone on some javascript feature and can merge them all back
    • stage/commit/branch/merge
    • stage is a temporary placeholder with your code
    • when you are happy with it you can commit and that is the specific save point
    • branch is working on a new feature
    • merge if the branch is approved
  • 3 buckets in git: unstaged/staged/commited

Setting up the Repository and making first commit

  • Login to Github and create new repo. Accept defaults
  • Open Git Bash and cd to project folder
  • initialize local directory as a git repo. Command: git init
  • git status will show the files which are at the moment unstaged
  • add the files in local repo and stage for commit. Command: git add .
  • git status should show the new files that are staged, now a commit can be used
  • first commit for the staged files. Command: git commit -m "First Commit". Now the first save point is made and git is tracking changes on those files
  • add an origin command: git remote add origin (paste github link here)
  • push branch to master branch in github. Command: git push origin master

Making additional Commits

  • git status checks what files have been modified
  • add the files to be staged git add . or -A.
  • commit files with a message git commit -m "your message"
  • make a pull request first git pull origin master
  • make a push request git push origin master

Make a new brach

  • create new branch: git branch new-branch
  • switch to it: git checkout new-branch. Output: "Switched to branch 'new-branch'"
  • stage all files: git add -A
  • commit with a message: git commit -m "Message" or if you want to use a defualt editor: git commit
  • to configure the default editor: git config --global core.editor "editor name(nano/vim)"
  • check status: git status
  • pull to the separate branch: git pull origin new-branch
  • push to the separate branch: git pull origin new-branch

Add to another remote server

  • create new remote server from the git project:git remote add upstream https://github.com/MiguelCabailo/WebDevEssentials.git
  • check the remote servers: git remote -v

Going back to a commit

  • git log --pretty=oneline
  • git checkout (comit hash)
  • git checkout -b (branch name, original for example for a branch without new changes)
  • always work on a branch so when checking out a previous commit create a branch to that so you can move back and forth

Make Changes to an experimental branch and merge to master

  • go to master branch
  • git merge (branch name)
  • merges that branch with the current branch now
